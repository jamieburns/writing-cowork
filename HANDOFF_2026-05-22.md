## Key decisions locked in this run

### Development workflow (settled)
- **Plugin dev cycle:** Claude Code CLI — `claude plugin marketplace update` then `claude plugin update writing-cowork@cowork-plugins-marketplace`.
- **Runtime:** Cowork (loads the same plugin from the marketplace).
- **Doc editing & discussion:** stays in Cowork chat. Only the two CLI commands at release time happen outside Cowork.
- **Marketplace structure:** Private plugin repo `jamieburns/writing-cowork`; public catalog repo `jamieburns/cowork-plugins-marketplace`.

### Consumer-side refresh (when Cowork update detection fails)
Only **marketplace-level uninstall + reinstall** invalidates Cowork's `remoteMarketplaceClient` proxy cache. Plugin-only operations do not. The 14-step UI procedure lives in `skills/pm-refresh-cowork-plugin/SKILL.md`.

### Issue logging mechanism (built v0.1.13)
- Specialists call `pm-create-issue-report` → drops cover-note in `<vault>/inbox/issues/`.
- Librarian processes via `pm-process-inbox-item` → dispositions: escalate / handle locally / close.
- Escalation: `pm-escalate-issue` uses `gh issue create --repo jamieburns/writing-cowork`, appends URL back to the cover-note, archives it to `process/history/`.
- Labels required on the repo: `severity:low`, `severity:medium`, `severity:high`, `from-vault`, `process-gap`, `skill-bug`, `documentation`, `enhancement`.

### Release checklist (memory: `feedback_release_includes_version_in_description.md`)
Every version bump must update **all three** places:
1. `.claude-plugin/plugin.json` → `version` field
2. `skills/pm-version/SKILL.md` → EXPECTED VERSION marker (two replace-all edits: description + output template)
3. **`description` suffix `(vX.Y.Z)`** in both `plugin.json` and the marketplace.json plugin entry — so the loaded version shows in Cowork's Browse-Plugins UI without opening cached files.
