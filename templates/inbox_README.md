# inbox/

Drop-zone for cross-context requests routed to the librarian (PM context).

Specialist contexts (substance, voice, reader-review, analysis, etc.) write cover-notes here; the librarian reads them via `pm-process-inbox-item` and applies the requested action.

## Subdirectories shipped by writing-cowork

- **`promotion/`** — artifact handoffs. A specialist has produced a deliverable (a piece of writing, an analysis output, an extracted excerpt) and wants the librarian to file it into the canonical project structure. The cover-note describes the artifact, its source, and where it should be placed. The artifact itself sits alongside the cover-note. Generated by `pm-create-promotion-request`. Processed by `pm-process-inbox-item`.
- **`hub-updates/`** — `project_hub.md` change requests. A specialist has noticed that the project hub's "where are we" summary is out of date (a milestone completed, a phase advanced, a status shifted). The cover-note describes the desired edit. Generated by `pm-create-hub-update-request`. Processed by `pm-process-inbox-item`.

Both subdirectories ship empty (`.gitkeep` placeholder). Cover-notes are added by the relevant skills; the librarian archives them to `process/history/` once processed.

## Vault-specific subdirectories

Vaults may add their own inbox conventions beyond what writing-cowork ships (for example, `external/` for incoming material from outside the project). Document those additions in this file as they emerge.
