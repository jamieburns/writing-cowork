## Workflow Summary

### Pull (before iPad work)
```
User → sync-mobile:setup (first time only)
     → sync-mobile:pull (before leaving Mac)
     → [Take iPad, work offline]
```

### Check (optional, from iPad)
```
iPad → dispatch sync-mobile:check-sync
    → Mac returns: vault current / behind
```

### Sync back (returning to Mac)
```
iPad → dispatch sync-mobile:sync-back
    → Mac detects changes
    → No conflict → merge + commit
    → Conflict → defer to temp branch + create summary
    ↓
    → User on Mac → sync-mobile:resolve-conflict
       → Choose strategy (iPad/Mac/manual)
       → Beyond Compare launches (if manual)
       → Commit + cleanup
```

## Data Structures

### mobile_scope.md
Defines which files iPad can edit. PM-maintained.

```markdown
# Mobile Editing Scope
## Writable directories/files
- ./substance/
- ./process/active/todos.md
- ./process/active/voice_exceptions.md
```

### sync_manifest.json
Created by `pull`, deleted after successful `sync-back`. Tracks pull state for change detection.

```json
{
  "created_at": "2026-05-23T14:30:22Z",
  "ipad_session_id": "ipad-0523-x7k9m2",
  "mac_git_head": {
    "commit": "abc1def...",
    "timestamp": "2026-05-23T14:30:00+0000",
    "tag": null,
    "branch": "main"
  },
  "writable_scope": ["substance/", ...],
  "vault_path": "/Users/...",
  "notes": ""
}
```

### sync_conflict_*.md
Generated by `sync-back` if conflict detected. Guides `resolve-conflict` skill. Deleted after resolution.

```markdown
# Sync Conflict — 2026-05-23 14:30
...conflict details, conflicted files, resolution options...
```

## Key Design Decisions

1. **iCloud handles file sync** — not git. Skills work with already-synced files.
2. **All git operations on Mac** — mobile has unreliable git support.
3. **Manifest-based change detection** — compare git HEAD at pull vs. now.
4. **Temp branch deferral** — conflicts don't block; user resolves on Mac when ready.
5. **Beyond Compare for merges** — configured as `git mergetool bcompare`.
6. **Session IDs** — track pull/sync cycles for context.
7. **PM controls writable scope** — prevents accidental edits to system files.
8. **Graceful error handling** — wait/retry for iCloud, flag missing changes, pause before overwriting.

## Testing Scenarios

### No conflict (happy path)
1. `setup` ✓
2. `pull` ✓
3. Edit file on "iPad"
4. `sync-back` → detect changes → merge → commit ✓
5. Verify manifest deleted, no temp branch left ✓

### Conflict (deferred resolution)
1. `setup` ✓
2. `pull` ✓
3. Edit file on "iPad"
4. Edit same file on Mac
5. `sync-back` → detect conflict → create temp branch + summary ✓
6. `resolve-conflict` → choose strategy → merge → commit ✓
7. Verify summary + branch deleted ✓

### Scope violation
1. `setup` ✓
2. Edit file outside mobile_scope.md on "iPad"
3. `sync-back` → detect violation → ask user ✓
4. User chooses: sync anyway or reject ✓

### Missing changes
1. `setup` ✓
2. `pull` ✓
3. Don't edit anything on "iPad"
4. `sync-back` → no changes detected → flag + ask debug ✓

### iCloud delay
1. `pull` ✓
2. Edit on "iPad" (but don't wait for iCloud)
3. `sync-back` → iCloud not synced yet → wait/retry → eventually syncs ✓

## Notes for Maintainer

- **Manifest is ephemeral:** Created at pull, deleted after sync. Don't commit it to git.
- **mobile_scope.md is permanent:** Track and commit. PM maintains it.
- **Conflict summaries are scratch files:** Helpful for user context, deleted after resolve.
- **Beyond Compare required:** Skills assume it's configured. Setup skill guides this.
- **iCloud is prerequisite:** This feature depends on iCloud vault sync. Won't work with pure git sync.
